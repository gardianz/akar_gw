TUTORIAL ROOTSFI BOT V11
========================


[1] INSTALL
-----------
Syarat: Node.js 18+ dan VPS Ubuntu 20+

  npm install


[2] JALANKAN
------------
  npm start

Pilih mode:
  1. internal     = kirim antar akun sendiri
  2. external     = kirim ke alamat di recipient.txt
  3. balance-only = cek saldo saja

OTP dari email hanya diminta pertama kali (session disimpan di tokens.json).


[3] ISI FILE INI DULU
----------------------

A) accounts.json

  {
    "accounts": [
      { "name": "akun1", "email": "a@gmail.com", "address": "roots...::1220..." },
      { "name": "akun2", "email": "b@gmail.com", "address": "roots...::1220..." }
    ]
  }

  - name    : label unik (bebas)
  - email   : email terdaftar di RootsFi
  - address : alamat Canton (wajib untuk mode internal)
  Minimal 2 akun untuk mode internal.


B) config.json — yang penting diatur:

  workers           = jumlah akun kirim paralel (mulai dari 3)
  minDelayTxSeconds = jeda MIN antar TX (detik)
  maxDelayTxSeconds = jeda MAX antar TX (detik)
  hourlyMaxTx       = max TX per jam per akun
  minHoldBalanceCc  = saldo CC minimum yang ditahan (tidak dikirim)
  tierAmounts       = nominal CC per kirim sesuai tier akun


C) recipient.txt — hanya untuk mode external, satu alamat per baris


[4] SETUP CAPTCHA SOLVER (WAJIB!)
----------------------------------
Bot butuh captcha solver. Ada 4 pilihan:

  A) SELF-HOSTED (GRATIS — DISARANKAN)
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Jalankan di VPS (bisa VPS yang sama atau VPS terpisah):

    sudo apt update -y && sudo apt install xvfb python3 python3-pip python3-venv -y
    cd Turnstile-Solver-main
    python3 -m venv venv
    source venv/bin/activate
    pip install fastapi==0.95.2 uvicorn "camoufox[fetch]" loguru psutil playwright
    python3 -m camoufox fetch
    python3 -m playwright install-deps
    sudo ufw allow ssh
    sudo ufw allow from [IP_ADDRESS_BOT] to any port 8000
    sudo ufw enable
    screen -S solver
    xvfb-run -a python3 api_server.py --yes &

  config.json:
    "captcha": {
      "provider": "self-hosted",
      "solverUrl": "http://IP_SOLVER:8000"
    }

  Jika solver di VPS LAIN, ganti localhost dengan IP VPS solver:
    "solverUrl": "http://IP_VPS_SOLVER:8000"

  Bisa pakai LEBIH DARI 1 solver (load balance otomatis):
    "solverUrl": [
      "http://IP_SOLVER_1:8000",
      "http://IP_SOLVER_2:8000"
    ]

  Tips: jalankan solver di screen agar tidak mati saat SSH putus:
    screen -S solver
    source venv/bin/activate
    xvfb-run -a python3 api_server.py --yes
    (Ctrl+A lalu D untuk detach, screen -r solver untuk kembali)

  B) 2CAPTCHA (BERBAYAR)
  ~~~~~~~~~~~~~~~~~~~~~~
  Cocok kalau tidak mau setup solver sendiri.

  config.json:
    "captcha": {
      "provider": "2captcha",
      "apiKey": "API_KEY_2CAPTCHA"
    }

  C) MULTIBOT (BERBAYAR)
  ~~~~~~~~~~~~~~~~~~~~~~
  Format mirip 2captcha, tapi pakai API key Multibot.

  config.json:
    "captcha": {
      "provider": "multibot",
      "apiKey": "API_KEY_MULTIBOT"
    }

  D) AUTO (SELF-HOSTED -> REMOTE FALLBACK)
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Bot akan coba self-hosted dulu. Kalau gagal, bot fallback ke provider remote.
  Default fallback = 2captcha. Bisa diganti ke multibot.

  config.json:
    "captcha": {
      "provider": "auto",
      "solverUrl": "http://IP_SOLVER:8000",
      "fallbackProvider": "multibot",
      "apiKey": "API_KEY_REMOTE"
    }

[7] TIPS
--------
  - Jalankan balance-only dulu untuk cek saldo & tier
  - Pastikan saldo cukup: minHoldBalanceCc + tierAmounts.max
  - File tokens.json = session login → JANGAN di-share
  - Session expired? Hapus tokens.json, jalankan ulang
  - workers besar = cepat tapi berat; mulai dari 3


[8] TROUBLESHOOTING
---------------------
  - "Self-hosted solver tidak bisa dihubungi"
    → Pastikan solver jalan: curl http://IP_SOLVER:8000/
    → Jalankan ulang: xvfb-run -a python3 api_server.py --yes

  - "Self-hosted solver penuh (429)"
    → Kurangi workers di config.json atau tambah solver

  - "Vercel Security Checkpoint"
    → Hapus tokens.json, jalankan ulang bot

  - Bot hang / tidak kirim
    → Ctrl+C, jalankan ulang


[9] STOP BOT
-------------
  Ctrl+C di terminal.
